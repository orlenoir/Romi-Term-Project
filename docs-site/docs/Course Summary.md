Summary:
Over the course of this class, our lab group gradually built up the software and hardware needed to run and control our Romi robot. At the beginning of the quarter, we focused on the basics of programming the Nucleo board using MicroPython. These early labs were mainly about learning how the microcontroller interacts with hardware such as motors, encoders, and timers. This gave us a foundation for understanding how embedded software controls real physical systems.

Once we were comfortable working with the hardware, we started organizing our code using cooperative multitasking. Instead of writing one large program, we split our system into multiple tasks that run alongside each other and share information when needed. We also implemented finite state machines to control how different parts of the program behave. This structure made our code easier to expand as we added more features throughout the quarter.

After establishing this structure, we began integrating sensors into the robot. Our group wired and programmed the line sensor and the bump sensors so the robot could detect its environment. The bump sensors were connected through GPIO pins and allow the robot to detect when it physically contacts an obstacle. The line sensor array was wired to read reflectance values from the ground so the robot can determine its position relative to a line. We then wrote code to read and interpret these signals inside our task system so the robot could react to them in real time.

Once the robot could sense its surroundings, we worked on improving its motion using closed loop motor control. Using feedback from the wheel encoders, we implemented a controller that regulates motor speed and allows the robot to reach and maintain a desired velocity. This step introduced us to control system concepts and required tuning the controller gains to achieve a stable and responsive step response.

Later in the project we incorporated the IMU to perform state estimation. The IMU provides measurements of acceleration and angular velocity, which can be used to estimate the robot’s orientation and movement over time. We added a task that reads the IMU data sequentially and processes it so the robot can estimate its heading and rotational motion while it drives. This information works alongside the encoder and line sensor data to give a better overall picture of the robot’s state.

To help analyze the performance of our control system, we also implemented a serial user interface and automated data collection system. Through the serial terminal we can enter controller gains, choose setpoints, and trigger step response tests. A Python script running on a computer communicates with the microcontroller, collects the output data, and automatically generates plots of the results for analysis.

The final term project ties all of these pieces together. Throughout the quarter we developed drivers, sensor interfaces, control algorithms, and a structured multitasking program. In the final project we combine the motors, encoders, IMU, line sensors, bump sensors, and all of the software we wrote to complete the course challenge. The result is a fully integrated robotic system that builds directly on each stage of the work we completed earlier in the class.
